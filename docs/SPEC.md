## 0. プロジェクトの目的とコンセプト

### 一行で
医学における「標準治療」の子育て版。**現在の科学的根拠（エビデンス）に基づき、多くの人にとって最も利益が大きいと考えられ、専門家の間で推奨されているやり方**を、妊娠期から独り立ちまで一貫して構造化し、誰でも無料で見られる形で公開する。

### 何を作るか
- 1つの推奨アクション = 1枚の「カード」。
- カードは **領域（10分類）** と **年齢タグ（8区分）** で整理され、**領域別ビュー**と**年齢別ビュー**の両方から引ける。
- 各カードには **エビデンス強度** と **影響度** の2軸評価が必ず付く。
- トップに「エビデンスA × 致命的リスク回避」のカードだけを集めた **絶対リスト（The Essentials）** を置き、コンセプトを一目で伝える。

### 何を作らないか（スコープ外・免責）
- 個別の医療相談・診断・治療の代替にはならない。緊急時・個別症例は必ず専門家へ。
- 「こうすれば必ず幸せになる」という保証はしない。あくまで**確率を上げる／致命的リスクを下げる**という確率的フレーム。
- 政治的・宗教的・文化的な価値観に踏み込む領域（例：宗教教育の是非）は「エビデンスで語れる範囲」に限定し、それを明記する。

### 設計思想（fladdict サイトから継承する原則）
1. **事実と解釈を分ける。** カードは「推奨内容（事実に近い）」と「解釈・注意（編集判断）」を視覚的に分離する。
2. **出典を必ず残す。** 一次資料（学会ガイドライン・公的機関・システマティックレビュー）へのリンクを各カードに最低1つ。
3. **育てる前提。** 「AI補助で作成／随時更新／間違いの指摘歓迎」を明記。各カードに更新日と、GitHub Issue への導線。
4. **正直であること自体が価値。** 根拠が弱い項目は「弱い」と正直に書く。俗説は「俗説」と明示する。これが最大の差別化。

## 2. カードのデータスキーマ（最重要）

各カードは `src/content/cards/<id>.md`。フロントマターは以下。Astro Content Collections の Zod スキーマで**厳密に検証**する（欠けたら build を失敗させる）。

```yaml
---
id: "safe-sleep-supine"                    # 一意なスラッグ（英小文字+ハイフン）
title: "赤ちゃんはあおむけで寝かせる"          # カードの見出し＝1つのアクション
statement: "毎回の睡眠で、平らで硬い面の上に、あおむけ（背中を下）で寝かせる。" # 推奨の要点1〜2文

# --- 分類 ---
domain: "safety-sleep"                      # 領域ID（§3参照。1つだけ）
ages: ["prenatal-postpartum", "0-1"]        # 該当年齢タグ（§4。複数可）

# --- 2軸評価（コンセプトの核） ---
evidence: "A"                               # A | B | C | observational （§5.1）
impact: "critical"                          # critical | serious | qol （§5.2）

# --- メタ ---
tags: ["SIDS", "睡眠環境", "窒息予防"]         # 自由タグ（横断検索用）
updated: "2026-07-05"                        # 最終更新日（YYYY-MM-DD）
status: "published"                         # draft | review | published
confidence_note: ""                         # エビデンスに議論・地域差がある場合の補足（任意）
---

## なぜ（背景）
<!-- 事実に近い記述。専門家向け表示で厚くなる部分。 -->

## 具体的にどうする（実践）
<!-- 本人向け表示で優先される、行動レベルの箇条書き。 -->

## よくある誤解・俗説
<!-- 「〜と言われるが根拠は弱い/否定されている」を明示。差別化の核。 -->

## 出典
<!-- 一次資料。機関名・タイトル・URL・（可能なら）発行年。最低1つ。 -->
- [機関名：ガイドライン名（年）](https://...)
```

### スキーマ検証ルール（Zod）
- `id` はファイル名と一致。全カードで一意。
- `domain` は §3 の enum のいずれか。
- `ages` は §4 の enum の配列（1つ以上）。
- `evidence` は `A|B|C|observational` のいずれか。
- `impact` は `critical|serious|qol` のいずれか。
- 本文に `## 出典` セクションが存在し、リンクが最低1つあることを、ビルド時スクリプトでチェック（なければ warning を出す）。

## 5. 2軸評価の定義（コンセプトの心臓部）

### 5.1 エビデンス強度（evidence）
子育て研究はRCTが倫理的・実務的に困難で、観察研究が中心。医学ガイドラインをそのまま当てはめず、**この現実に合わせた4段階**にする。カードには「なぜこの評価か」が分かる短い注記を推奨。

| 値 | ラベル | 意味 | 目安となる根拠 |
|---|---|---|---|
| `A` | 強い根拠 | 複数の質の高い研究／システマティックレビューで一貫。主要機関が明確に推奨。 | コクランレビュー、AAP/WHO/NICEの強い推奨、複数RCT |
| `B` | 中程度の根拠 | 良質な研究はあるが数が限られる、または一部に不一致。専門家は概ね支持。 | 少数のRCT、質の高いコホート研究の一致 |
| `C` | 弱い／限定的な根拠 | 観察研究中心で交絡の可能性、または専門家の意見が主。方向性は示唆される。 | 単発の観察研究、専門家コンセンサス |
| `observational` | 相関どまり | 相関は報告されるが因果は不明。過度な断定を避けるべき領域。 | 相関研究のみ、メカニズム仮説段階 |

> **重要な編集方針：** 「やってはいけない（致命的リスク回避）」系はエビデンスAが集まりやすく、「こうするとより良く育つ」系は C〜observational が多い。この非対称性を隠さず、むしろ設計上の売りにする。カードUIでエビデンスAとobservationalが視覚的に一目で区別できること。

### 5.2 影響度（impact）
| 値 | ラベル | 意味 |
|---|---|---|
| `critical` | 致命的リスクに関わる | 守らないと死亡・重篤・不可逆な障害の恐れ（SIDS、窒息、溺水、チャイルドシート等） |
| `serious` | 重大な影響 | 健康・発達・安全に大きく関わるが、直ちに致命的ではない（予防接種の一部、虐待防止、栄養等） |
| `qol` | 生活の質・より良い育ち | 守れれば望ましいが、致命性は低い（読み聞かせ、遊び、非認知能力等） |

### 5.3 絶対リスト（The Essentials）の定義
`evidence == "A" && impact == "critical"` のカードを自動抽出してトップに並べる。「最低限これだけは」という命綱リスト。**ここが標準治療コンセプトの一番の見せ場**。

## 6. デザイン方針（デザイントークン）

読み手が「本人／専門家」を切り替える前提。落ち着いていて、医療情報として信頼でき、かつ冷たすぎない。fladdict版のカード陳列は踏襲しつつ、色とタイポで独自性を出す。

### コンセプト
「診療ガイドラインの信頼感」×「親に寄り添う温度」。**カルテのような余白と罫線の規律**の中に、年齢の流れを感じさせる縦のタイムライン。派手さは1点（エビデンスバッジの色分け）に集中させ、他は静かに保つ。

### カラートークン（CSS変数）
```css
:root {
  --bg:            #FBFAF7;  /* 紙のようなオフホワイト */
  --surface:       #FFFFFF;  /* カード面 */
  --ink:           #1F2421;  /* 本文（真っ黒を避けた濃い墨色） */
  --ink-soft:      #55605A;  /* 補助テキスト */
  --line:          #E4E2DA;  /* 罫線・区切り */
  --accent:        #2E6F6A;  /* 深い緑（信頼・落ち着き。医療の青緑系） */

  /* エビデンス強度バッジ（唯一の"派手さ"の置き場所） */
  --ev-a:          #1B7F5A;  /* 強い根拠：濃い緑 */
  --ev-b:          #6FA03C;  /* 中程度：黄緑 */
  --ev-c:          #C9A227;  /* 弱い：くすんだ金 */
  --ev-obs:        #9AA0A6;  /* 相関どまり：グレー */

  /* 影響度バッジ */
  --impact-critical: #B23A48; /* 致命的：抑えた赤 */
  --impact-serious:  #C77D33; /* 重大：琥珀 */
  --impact-qol:      #5B7C99; /* QOL：青灰 */
}
```
> 注意：Anthropic系の terracotta（#D97757 付近）やクリーム地×高コントラストserifの「いかにもAI生成」な組み合わせは避ける。緑基調は「健康・安心」の含意があり子育てテーマと整合する。

### タイポグラフィ
- **見出し（display）：** 「Zen Old Mincho」等の日本語明朝系を要所に。ガイドライン文書の格を出す。使いすぎない。
- **本文（body）：** 「Noto Sans JP」。可読性最優先。
- **データ・バッジ（utility）：** 等幅または「IBM Plex Mono」等で、エビデンス記号（A/B/C）を記号らしく見せる。
- 型スケールを明示（例：本文16px / 行間1.8、見出しは1.5〜2.0rem）。

### シグネチャー要素
**「エビデンス×影響度」の2軸ミニマトリクス。** 各カードの右上に小さな2×2のドット表示を置き、そのカードがマトリクスのどこに位置するかを一目で示す。一覧ページでは全カードをこのマトリクス上にプロットした俯瞰ビューも用意（散布図的）。これがサイトの記憶に残る顔になる。

### アクセシビリティ（品質の床）
- モバイルまで完全レスポンシブ。
- バッジは色だけでなく必ずラベル文字（A/B/C、致命的/重大/QOL）を併記（色覚多様性対応）。
- キーボードフォーカスが見えること。`prefers-reduced-motion` を尊重。

## 7. 画面構成

### 7.1 トップページ `/`
1. **ヒーロー：** サイトの一行説明（「子育ての地図」）＋ 強い免責への導線。派手なグラデや巨大数字は使わない。診療ガイドラインの表紙のような静けさ。
2. **The Essentials（絶対リスト）：** `evidence:A × impact:critical` のカードを大きめに陳列。「まずこれだけは」。
3. **2つの入口：** [年齢から見る] / [領域から見る] の大きな2ボタン。
4. **俯瞰マトリクス：** 全カードを2軸にプロットしたミニ図（クリックで該当カードへ）。
5. **このサイトについて／免責／更新方針** への導線。

### 7.2 年齢別ビュー `/age/<ageId>/`
- 縦タイムライン。妊娠期→独り立ちの流れを視覚化。
- 選んだ年齢に該当するカードを、影響度（critical→serious→qol）順に並べる。

### 7.3 領域別ビュー `/domain/<domainId>/`
- 10領域それぞれの一覧。領域内でエビデンス強度・影響度でソート／フィルタ可能。

### 7.4 カード詳細 `/card/<id>/`
- 見出し・statement・2軸バッジ・年齢タグ。
- **本人向け / 専門家向け トグル**（§8）。
- 「なぜ」「どうする」「よくある誤解」「出典」。
- 更新日、Issueで指摘するリンク。

### 7.5 固定ページ
- `/about/`：目的・作り方・限界・AI補助の明示。
- `/methodology/`：エビデンス分類基準（§5をそのまま公開）。透明性の証。
- `/disclaimer/`：医療免責（§9）。

## 8. 「本人向け／専門家向け」切替の実装方針

同一カード内に両方の情報を持たせ、**表示の粒度を切り替える**（別ページにはしない）。

- **実装：** ページ上部のトグル（状態は `localStorage` **ではなく** URL クエリ or Astro の client-side な小さなスクリプトで保持。※Artifact制約とは無関係、GitHub Pages上なので localStorage 使用可。ただし初期表示は「本人向け」をデフォルトに）。
- **本人向け（default）：** `## 具体的にどうする` と `## よくある誤解` を前面に。専門用語は最小限、結論先行。
- **専門家向け：** `## なぜ（背景）` の詳細、エビデンスの限界・議論（`confidence_note`）、出典を全面展開。研究名・機関の推奨強度まで見せる。
- MDX で `<ForExpert>...</ForExpert>` / `<ForParent>...</ForParent>` のようなコンポーネントで囲い、トグルで表示切替できるようにするのが簡潔。

## 9. 免責・編集ポリシー（サイトに必ず明記）

- 本サイトは一般的な情報提供であり、**個別の医療・心理・法的アドバイスではない**。診断・治療・緊急対応は必ず専門家・医療機関へ。
- 推奨は「多くの人にとって利益が大きいと考えられる標準的な考え方」であり、**すべての子・家庭に当てはまるとは限らない**。子には個人差があり、既往・障害・家庭環境により最適解は変わる。
- エビデンスは更新される。**掲載時点の理解**であり、最新の学会見解と異なる場合がある。更新日を確認のこと。
- **AI（Claude）による調査補助で作成**している。誤り・古い情報が含まれうる。各カードの出典を確認し、疑問は GitHub Issue へ。
- 出典は可能な限り一次資料（学会・公的機関・システマティックレビュー）を用いる。二次情報のみの項目はその旨を明記する。

## 10. コンテンツ制作の運用フロー（人間＋AIの分担）

カードは以下の状態遷移で管理（`status` フロントマター）。
1. `draft` … トピック立てのみ、または下書き。
2. `review` … 本文と出典が揃い、レビュー待ち。
3. `published` … 公開。トップ・一覧に表示。

**1カードの作り方テンプレ（AIに調査させる際のプロンプト骨子）：**
> 「〈トピック〉について、AAP・WHO・NICE・コクラン・日本の公的機関（こども家庭庁・厚労省・日本小児科学会等）の一次資料を調べ、(1)推奨の要点、(2)背景、(3)実践手順、(4)よくある誤解、(5)出典URL を整理。エビデンス強度をA/B/C/observationalで判定し、判定理由を一文で。影響度をcritical/serious/qolで判定。」

**優先制作順（推奨）：**
1. `evidence:A × impact:critical`（絶対リスト）を全領域から先に埋める＝サイトの背骨。
2. 妊娠期〜1歳（相談・不安が最も多い時期）。
3. 以降、年齢を追って各領域を拡張。

## 付録A：サンプルカード（スキーマの実物見本）

> ※内容の正確性は別途この後に精査版を作成する。ここでは**スキーマの形**を示すためのサンプル。

id: "safe-sleep-supine"
title: "赤ちゃんはあおむけで寝かせる"
statement: "1歳になるまで、毎回の睡眠で、平らで硬い面の上に、あおむけ（背中を下）で寝かせる。"
domain: "safety-sleep"
ages: ["prenatal-postpartum", "0-1"]
evidence: "A"
impact: "critical"
tags: ["SIDS", "睡眠環境", "窒息予防"]
updated: "2026-07-05"
status: "published"
confidence_note: ""

## なぜ（背景）
うつぶせ・横向き寝は乳幼児突然死症候群（SIDS）のリスクを高めることが、
複数の国の疫学研究とその後の「あおむけ寝キャンペーン」による死亡率低下で
一貫して示されている。主要な小児科学会・公的機関が一致して推奨している。

## 具体的にどうする（実践）
- 昼寝も夜も、毎回あおむけで寝かせる。
- 面は硬く平らに。やわらかい寝具・枕・ぬいぐるみは寝床に置かない。
- 寝返りができるようになっても、寝かせるときはあおむけから始める。

## よくある誤解・俗説
- 「吐いたものを詰まらせるからあおむけは危険」→ 健康な乳児では根拠が弱く、
  あおむけが誤嚥を増やすという明確な証拠はないとされる。
- 「頭の形が悪くなるから横向きに」→ 向き癖対策は覚醒時のうつぶせ遊び等で、
  睡眠姿勢をうつぶせにする理由にはならない。

- [AAP: Safe Sleep / 2022 Policy Statement](https://www.aap.org/en/patient-care/safe-sleep/)
