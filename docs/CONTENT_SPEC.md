# CONTENT_SPEC.md — カード制作・データ参照仕様書

**バージョン：v1.2（2026-07-05）**

> **目的：** 誰が（どのAIモデルが）作業しても、同じ品質のカードができるようにする。
> そのために、判断を「センス」に任せず、**手順・決定木・チェックリスト**にすべて落とし込む。
> 本書は SPEC.md（サイト実装仕様）の §2 カードスキーマとセットで使う。

> **変更履歴：**
> - v1.2：実カード2枚（はちみつ／あおむけ寝）の試作で判明した穴を反映。①原本URLがfetch不可な場合の代替手順（§3）②confidence_noteの3つの型と200字上限（§8.5）③加熱で無効化されない禁忌のstatement注記（§8）④通達・事務連絡もTier1に含む＆AAPのエビデンスグレードを判定に利用（§3）⑤調査ログにグレード転記欄を追加（§5）⑥セルフチェックに4項目追加（§10）。
> - v1.1：fetch検証要件・セクション別根拠要件・出典年表記を追記。

## 0. 三大原則（すべてに優先する）

1. **記憶で書かない。** すべての事実・数値・推奨は、この作業中に実際に開いて確認した情報源に基づく。モデルが「知っている」ことは出典にならない。
2. **開けなかったものは存在しない。** URLを実際に取得（fetch）し、該当の記載を目視確認できたソースだけが出典になれる。検索結果のスニペットだけを根拠に引用してはならない。
3. **迷ったら下げる・書かない。** エビデンス判定に迷ったら低い方に。出典が見つからない主張は書かない。判断できない場合はカードを完成させず、TODOを残して止まる。**「不完全だが正直」は合格、「完全だが盛った」は不合格。**

## 1. 制作パイプライン（この順番以外で作業しない）

```
STEP 1  トピック確定       … 1カード=1アクションに切る（§2）
STEP 2  検索クエリ設計     … 日英2言語×機関名込み（§4）
STEP 3  ソース収集         … Tier1を最優先で最低2本（§3）
STEP 4  URL検証            … 全URLをfetchし該当記載を確認（§0-2）
STEP 5  調査ログ作成       … §5のフォーマットを完全に埋める
STEP 6  エビデンス判定     … §6の決定木に機械的に従う
STEP 7  影響度判定         … §7の決定木に機械的に従う
STEP 8  カード執筆         … §8の執筆ルールに従う
STEP 9  セルフチェック     … §10のチェックリスト全項目
STEP 10 status設定         … 全項目✓なら review、1つでも✗なら draft
```

**停止条件（ここに該当したら先に進まず、TODOを書いて止まる）：**
- Tier1ソースが1本も見つからない
- ソース間で推奨の方向が真逆（片方が推奨、片方が非推奨）
- 日本と海外で公的推奨が大きく異なり、どちらを主とすべきか判断できない

## 2. STEP 1：トピックの切り方

- **1カード = 親が取れる1つのアクション。**「睡眠の安全」は大きすぎる（分割する）。「あおむけで寝かせる」「寝床に物を置かない」は適切な粒度。
- タイトルは動詞で終わる行動文にする（例：「〜を与えない」「〜で寝かせる」「〜を受ける」）。
- 知識だけで行動がないもの（例：「SIDSとは何か」）はカードにしない。用語解説は各カードの「なぜ」内で行う。

## 3. 情報源のルール（ソース階層）

### Tier 1 — 一次推奨元（最優先。カードの根拠の主軸）
| 種別 | 具体例（ドメイン） |
|---|---|
| 国際機関 | WHO（who.int） |
| 日本の公的機関 | こども家庭庁（cfa.go.jp）、厚生労働省（mhlw.go.jp）、消費者庁（caa.go.jp）、政府広報（gov-online.go.jp） |
| 日本の学会 | 日本小児科学会（jpeds.or.jp）、日本産科婦人科学会（jsog.or.jp）、日本周産期・新生児医学会 等 |
| 海外の学会・公的機関 | AAP（aap.org / publications.aap.org）、CDC（cdc.gov）、NICE（nice.org.uk）、NHS（nhs.uk）、USPSTF（uspreventiveservicestaskforce.org） |
| システマティックレビュー | コクラン（cochranelibrary.com / cochrane.org） |

**Tier1の範囲について（v1.2 追記）：**
- 上記機関の**公開政策ページだけでなく、同機関が発出する通達・告示・事務連絡・プレスリリースもTier1に含む**（例：厚生労働省の事務連絡、こども家庭庁のプレスリリース）。特に「実際の死亡・事故事例の公式記録」はこれらの文書にしかないことが多く、critical判定の裏付けとして重要。
- **AAPガイドラインは各推奨に独自のエビデンスグレード（A/B/C）を明記している。** これは本プロジェクトのエビデンス分類（§6）とほぼ対応するため、AAP由来の推奨は**AAP自身のグレードを判定の第一根拠として利用してよい**（例：AAPが「A」としている推奨は、本プロジェクトでもAの有力な根拠になる）。調査ログにAAPのグレードを転記すること。NICE・USPSTFなども同様に推奨度・エビデンスレベルを明示していることが多い。

### Tier 2 — 補強用（Tier1の内容を補足・具体化する用途のみ）
- 大学病院・主要医療機関の患者向け情報（Mayo Clinic、国立成育医療研究センター等）
- 自治体の公式ページ（〜.lg.jp）※国の一次資料の要約として
- 査読付き個別論文（PubMed収載）※単独でA判定の根拠にはしない

### Tier 3 — 使用禁止（検索上位に出ても出典にしない）
- 商業メディア・企業ブログ（育児グッズメーカー、EC付随ブログ等）
- まとめサイト、個人ブログ、SNS、動画
- ニュース記事（一次資料への到達手段としてのみ利用可。出典欄には書かない）

### 本数の要件
- 全カード共通：**出典は最低2本、うちTier1が最低1本。**
- `impact: critical` のカード：**Tier1が最低2本**（日本の公的機関1本＋海外機関または学会1本が理想）。

### fetch検証の要件（v1.1 追記）
- **出典欄に載せるURLは、原則すべて実際にfetchし該当記載を目視確認したもの。**
- どうしても未fetchのソースを載せる場合（例：検索スニペットのみ確認）は、
  (a) そのソースを**カード本文の主張の主たる根拠にしない**、
  (b) 調査ログに「fetch未実施・補強扱い」と明記する、
  の両方を満たすこと。critical カードでは未fetchソースを主たる根拠にしてはならない。

### 原本URLがfetchできない場合の手順（v1.2 追記・重要）
学術誌・一部の海外機関（例：AAPのpublications.aap.org）は、bot検出により原本URLのfetchがブロックされることがある。**この場合でもそのソースを諦めず、以下の順で内容検証を試みる。**

1. **同一機関が別ドメインで公開している全文**（例：機関のトップドメイン側のPDF）をfetch。
2. **大学・公的機関・自治体がホストしている、その資料の公式PDF全文の転載**をfetch（例：`〜.edu` や `〜.go.jp`、州保健局サイトにあるAAPガイドラインPDF）。転載元が信頼できる機関であることを確認する。
3. 上記でガイドライン本文の該当記載を確認できたら、**内容検証は完了**とみなす。

**出典欄の書き方：** リンクは**読者が参照すべき正規の原本URL**（例：AAP公式ページ）を記載する。ただし調査ログには**「原本はfetch不可。内容検証は〈転載元〉のPDFで実施」と必ず明記**し、実際にfetchしたURLも残す。これにより「読者向けの正しいリンク」と「検証の透明性」を両立する。

**禁止：** 転載PDFすら見つからず、検索スニペットだけしか確認できていないのに、そのソースを主たる根拠として使うこと。その場合は上記「未fetchソース」の扱い（補強扱い・主根拠にしない）に従う。

### セクション別の根拠要件（v1.1 追記）
- 「具体的にどうする」「よくある誤解」に書く各主張も、**いずれかの出典に対応する記載が必要**。
  出典にない実践手順・俗説否定を書かない（例：「加熱では死なない」「母乳では感染しない」も出典必須）。
- 調査ログの「確認した記載」に、これらの主張の裏付けを含めること。

### 出典の年表記（v1.1 追記）
- 出典リンクには、**発行年または最終更新年を可能な限り併記する**（例：`（2026年4月更新）`）。
- 年が確認できない場合は無理に書かず、調査ログにその旨を残す。鮮度の判断材料を読者に渡すことが目的。

## 4. STEP 2：検索クエリの設計

- **必ず日英2言語で探す。** 日本語：`<トピック> こども家庭庁` `<トピック> 厚生労働省` `<トピック> 日本小児科学会`。英語：`<topic> AAP recommendation` `<topic> WHO guideline` `<topic> Cochrane review`。
- クエリは1〜6語で短く。機関名を入れると一次資料に直接届きやすい。
- 発行年を確認し、**同一機関の新しい版があれば必ず新しい方**を使う（例：旧ガイドラインの検索結果に注意）。
- 検索結果の上位に Tier 3（商業ブログ）が並ぶことが多い。**無視して Tier 1 のドメインを探す。**

## 5. STEP 5：調査ログ（カードを書く前に必ず作る中間成果物）

調査ログなしにカード本文を書き始めることを禁止する。ファイルは `research/<card-id>.md` に保存し、カードと一緒にコミットする（監査可能にする）。

```markdown

作成日: YYYY-MM-DD / 作成モデル: <モデル名>

## 使用した検索クエリ
- "..."（日本語）
- "..."（英語）

## 検証済みソース（fetchして該当記載を確認したもののみ）
1. [Tier1] 機関名｜資料タイトル｜URL｜発行・更新年
   該当記載の要約（自分の言葉で1〜2文）: ...
   ※原本がfetch不可だった場合はここに明記：「原本fetch不可。検証は〈転載元・実際にfetchしたURL〉で実施」（§3の手順）
   ※ソースがエビデンスグレードを明示している場合（AAP等）はここに転記：「AAPグレード＝A」
2. [Tier1] ...
3. [Tier2] ...

## ソース間の一致・不一致
- 一致点: ...
- 不一致点: ...（なければ「なし」と明記）

- ...（なければ「なし」と明記）

- 判定: A / B / C / observational
- 決定木のたどり: Q1=Yes → Q2=No → A（§6の質問番号で示す）
- ソース側のグレード（あれば）: 例「AAP＝A、PrevInfad＝A」

- 判定: critical / serious / qol
- 根拠: ...
```

## 6. STEP 6：エビデンス強度の決定木（裁量禁止・上から順に）

以下の質問に**順番に**答える。最初に確定した時点で終了。

```
Q1. コクラン等のシステマティックレビュー／メタ分析が効果を支持している、
    または Tier1 機関の2つ以上が同じ方向で「明確に推奨」している。
    → YES → Q2 へ ／ NO → Q3 へ

Q2. 主要な Tier1 機関の間で、推奨の方向に不一致があるか？
    → NO  → 【A】
    → YES → 【B】＋ confidence_note に不一致の内容を必ず記載

Q3. Tier1 機関の1つが推奨している、または複数のRCTが存在する。
    → YES → 【B】 ／ NO → Q4 へ

Q4. 質の高い観察研究（コホート等）が複数、同じ方向で一致している、
    または Tier1/Tier2 の専門家コンセンサス文書がある。
    → YES → 【C】 ／ NO → Q5 へ

Q5. 相関を示す研究はあるが、因果の裏付け・機関の推奨がない。
    → YES → 【observational】
    → NO  → カード化を中止（根拠不足）。TODOを残して停止。
```

**補足ルール：**
- 「推奨の強さ」の読み方：ガイドライン内の "should / strong recommendation / 推奨する" は明確な推奨。"may consider / できれば / 望ましい" は弱い推奨（Q3〜Q4側）。
- 判定に1分以上迷ったら、**1段階下げて** confidence_note に迷った理由を書く。

## 7. STEP 7：影響度の決定木（上から順に）

```
Q1. 守らなかった場合、死亡または不可逆な重篤障害
    （溺水・窒息・SIDS・重度外傷・重い先天障害等）に直結しうるか？
    その裏付け（死亡統計・事故報告・症例報告）がソースにあるか？
    → YES → 【critical】

Q2. 入院レベルの健康被害、または発達・心身への長期的な悪影響
    （虐待・重い感染症・深刻な栄養不良等）につながりうるか？
    → YES → 【serious】

Q3. それ以外（より良い発達・生活の質・親子関係の向上等）
    → 【qol】
```

- critical は「理論上ありうる」ではなく、**ソースに死亡・重篤例への言及があること**を要件とする。

## 8. STEP 8：執筆ルール（セクション別）

共通文体：です・ます調。専門用語は初出時に一言説明。**逐語コピー禁止**（すべて自分の言葉で要約。原文の言い回しをそのまま写さない）。

| セクション | 分量目安 | ルール |
|---|---|---|
| `title` | 〜25字 | 行動文（動詞で終える） |
| `statement` | 1〜2文 | 「いつ・何を・どうする」が入った要点。数値（月齢等）は出典にあるもののみ。**禁忌食材・物質で「加熱・調理しても無効化されない」性質がある場合は、statementにその旨を必ず入れる**（例：はちみつ＝「加熱しても菌は死なない」）。読者が誤って「加熱すれば安全」と解釈するのを防ぐため（v1.2）。 |
| なぜ（背景） | 150〜400字 | 根拠の説明。専門家向け表示で読まれる前提で、研究・推奨の出どころに触れる |
| 具体的にどうする | 箇条書き3〜6点 | 各1文。今日から実行できる粒度。曖昧語（「適切に」「なるべく」）を避ける |
| よくある誤解・俗説 | 1〜3項目 | 「よく言われること → 実際の根拠の状態」の型で書く。俗説の否定にも出典が要る |
| `confidence_note` | 〜200字 | エビデンスに留保・議論・限界がある場合のみ記載（なければ空でよい）。**§8.5の3つの型のいずれかに沿って簡潔に書く**。200字を超えそうなら要点に絞る（v1.2） |
| 出典 | 2本以上 | `[機関名：資料名（年）](URL)` の形式。調査ログで検証済みのURLのみ |

**禁止表現：**
- 断定の過剰：「必ず」「絶対に安全」「100%」
- 医療判断の代替：「受診は不要」「薬をやめてよい」
- 恐怖煽り：「〜しないと死にます」（リスクは淡々と数字・事実で書く）
- 出典にない数値・固有名詞・年号

### 8.5 confidence_note の書き方（v1.2 追記）

confidence_note は「このカードの推奨をどのくらい強く信じてよいか」の但し書き。**留保がなければ空でよい**。書く場合は以下の3つの型のいずれかに当てはめ、200字以内に収める。作業者ごとの表現ブレを防ぐため、型と文例を固定する。

**型①：不一致型**（機関間・研究間で推奨や結論が割れている）
> 例：「〇〇（機関A）は△△を推奨するが、□□（機関B）は◇◇の立場をとる。この点は専門家の間で見解が分かれており、確立した結論ではない。」

**型②：地域差型**（日本と海外、または地域で基準・推奨が異なる）
> 例：「本カードは日本の〇〇基準に基づく。海外では△△が一般的で、数値・時期が異なる。居住地の公的推奨を優先のこと。」

**型③：有効性限定型**（推奨は妥当だが、守っても完全には防げない／根拠が相関ベース）
> 例：「〇〇の根拠は観察研究に基づく相関で、因果は完全には確立していない。守ることでリスクは下がるが、発症・事故をゼロにできるわけではない。」

- 複数の型が該当する場合は、最も重要なものを1つ選ぶ（詰め込まない）。
- `impact: critical` かつ `evidence: A` でも、型③（守っても完全には防げない）を要する項目は多い（例：SIDS対策）。「Aで重大」だからといって「守れば安全」と誤読させないため、限界を正直に書く。
- confidence_note は読者を不安にさせるためではなく、**過信も過小評価も避けてもらうため**に書く。トーンは中立・簡潔に。

## 9. 日本文脈の取り扱い

- **日本の公的推奨と海外の推奨が両方ある場合：** 本文は日本の推奨を主、海外を補強として書く。
- **海外にしか推奨がない場合：** その旨を明記する（「日本の公的機関の明示的な推奨は確認できなかった」）。confidence_note にも記載。
- **数値基準が国で異なる場合（例：ワクチンスケジュール、栄養基準）：** 日本の基準を採用し、異なることを一言添える。
- 添い寝・入浴習慣など**日本特有の生活文化に関わる項目**は、海外推奨の直訳をせず、日本の公的資料での言及を必ず探す。

- [ ] （v1.2）加熱・調理で無効化されない禁忌の場合、その旨を statement に入れた
- [ ] （v1.2）critical×A でも「守れば完全に安全」と誤読させていないか（限界を書いたか）

1つでも✗ → `status: draft` のまま、末尾に `<!-- TODO: ... -->` を残す。**✗を隠して review にすることを最大の違反とする。**

## 12. 品質のばらつきを検知する運用（レビュー側の手順）

人間（またはレビュー役のモデル）は、カード本文を読む前に**調査ログだけ**を見て以下を確認する：
1. 出典URLを2本抜き打ちで開き、要約が実際の記載と一致するか
2. 決定木のたどりが §6・§7 のルール通りか
3. 数値を1つ選び、出典内に存在するか

この3点が通らないカードは、本文の出来に関わらず差し戻す。**調査ログが信頼できないカードは、本文がどれだけ流暢でも不合格**——これが低品質モデル対策の最後の砦。
