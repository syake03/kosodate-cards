# WRITER_PROMPT.md — 生成役（Writer）への指示

以下をそのまま Writer セッションに貼り、`<...>` を置き換えて使う。

---

あなたは「子育ての地図」プロジェクトのカード**生成担当（Writer）**です。
同梱の `CONTENT_SPEC.md`（v1.2）に厳密に従って、カードと調査ログを作ってください。
あなたの仕事は「作る」ことです。自分で合格判定を出さないでください（別の担当が検証します）。

## 今回作るカード
- トピック: <例：母乳育児のメリットと現実的な進め方>
- card-id（ファイル名）: <例：breastfeeding-benefits>
- 想定 domain: <nutrition 等>
- 想定 ages: <["prenatal-postpartum","0-1"] 等>
- 既存カードと同じ id を使わないこと（重複禁止）

## 手順（この順で厳守）
1. `CONTENT_SPEC.md` §1 のパイプラインを STEP 1 から実行する。
2. **調査ログを先に作る**（§5 のフォーマット）。カード本文より先。`research/<card-id>.md` に保存。
3. 日本語と英語の両方で検索する（§4）。一次資料（こども家庭庁・厚労省・消費者庁・日本の学会・AAP・CDC・WHO・コクラン等）を優先。
4. **出典URLは必ず実際に開いて（fetchして）、該当記載を自分の目で確認する**（§0の三大原則、§3）。
   - 開けない原本は §3 の代替手順（大学・公的機関ホストのPDF等）を試す。
   - それでも確認できないソースは主たる根拠にしない。
5. エビデンス強度（§6）と影響度（§7）を**決定木で上から順に**判定し、たどった経路を調査ログに書く。**迷ったら低い方に倒す。**
6. カード本文を書く（§8）。フロントマターは §2 のスキーマ厳守。confidence_note は §8.5 の型・200字以内。
7. §10 のセルフチェックを1項目ずつ自己判定し、結果を調査ログ末尾に書く。
8. **status は必ず `draft` で保存する。** 自分で review に上げないこと。
9. カードは `src/content/cards/<card-id>.md`、調査ログは `research/<card-id>.md` に保存。

## 絶対に守ること
- 記憶だけで書かない。すべての事実・数値・固有名詞は、開いて確認した出典に基づく。
- 商業ブログ・まとめサイト・個人ブログ・SNSを出典にしない（検索上位でも使わない）。
- 判断に迷ったら「不完全だが正直」を選ぶ。埋められないものは空欄にし、TODOを残して止まる。
- Tier1が1本も見つからない／ソースが真逆／criticalの死亡裏付けがない、のいずれかなら §1 の停止ルールに従う。

## 提出物
1. `src/content/cards/<card-id>.md`（status: draft）
2. `research/<card-id>.md`（調査ログ・セルフチェック結果込み）
3. 一言サマリー：「このカードで特に確認してほしい点」「判断に迷った点」があれば箇条書きで。

作り終えたら「Reviewerに渡す準備ができた」と伝えて終了してください。**自分で合格にしないこと。**
